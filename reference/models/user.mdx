`ApplicationUser` extends `IdentityUser` from ASP.NET Core Identity, adding clinic-specific fields for profile information and the credit balance. All standard Identity fields — `Id`, `Email`, `UserName`, `PasswordHash`, `PhoneNumber`, `LockoutEnabled`, and others — are inherited unchanged from `IdentityUser`.

## Custom fields

These fields are defined directly on `ApplicationUser` and are stored as additional columns in the `AspNetUsers` table alongside the Identity columns.

<ResponseField name="FirstName" type="string" default='""'>
  User's first name. Defaults to an empty string.
</ResponseField>

<ResponseField name="LastName" type="string" default='""'>
  User's last name. Defaults to an empty string.
</ResponseField>

<ResponseField name="ImageFileName" type="string" default='""'>
  File name of the user's profile image, stored under `wwwroot/img/users/`. Defaults to an empty string when no image has been uploaded.
</ResponseField>

<ResponseField name="CreatedAt" type="DateTime">
  Timestamp of when the user account was created.
</ResponseField>

<ResponseField name="CreditosDisponibles" type="int" default="0">
  Current credit balance available to the patient. Validated with `[Range(0, int.MaxValue)]` — cannot go negative. Defaults to `0`. Incremented when a credit package is assigned via a `TransaccionCredito` and decremented when an appointment is booked. Non-patient roles hold this field at `0` by convention.
</ResponseField>

<ResponseField name="Horarios" type="ICollection<Horario>" default="new List<Horario>()">
  Navigation collection of availability schedules belonging to this user. Used for physiotherapists to define their weekly working hours. See [Credit and schedule models](/reference/models/credit) for the `Horario` model definition.
</ResponseField>

## Inherited Identity fields

`ApplicationUser` inherits all standard `IdentityUser` properties. The most commonly referenced ones are:

| Property | Type | Description |
|---|---|---|
| `Id` | `string` | GUID-based primary key generated by Identity. |
| `UserName` | `string` | Login username, typically set to the email address. |
| `Email` | `string` | User's email address. |
| `NormalizedEmail` | `string` | Uppercase email used for case-insensitive lookups. |
| `PasswordHash` | `string` | Bcrypt-hashed password managed by Identity. |
| `EmailConfirmed` | `bool` | Email verification flag. Set to `false` by default; `RequireConfirmedAccount` is disabled in `Program.cs`. |
| `PhoneNumber` | `string` | Optional phone number. |
| `LockoutEnabled` | `bool` | Whether account lockout on failed logins is active. |
| `AccessFailedCount` | `int` | Count of consecutive failed login attempts. |

## Roles

Four roles are seeded into the `AspNetRoles` table during `OnModelCreating`. Every user must be assigned one of these roles to access role-protected pages.

| Role | Normalized name | Description |
|---|---|---|
| `Admin` | `ADMIN` | Full system access. Manages users, services, credit packages, and all appointments. |
| `Recepcionista` | `RECEPCIONISTA` | Manages appointment booking, patient check-in, and credit assignment on behalf of patients. |
| `Fisioterapeuta` | `FISIOTERAPEUTA` | Views their own schedule and assigned appointments; adds clinical notes. |
| `Paciente` | `PACIENTE` | Books appointments using their own credit balance; views their appointment history. |

<Tip>
  The first admin account must be assigned the `Admin` role manually through the database after initial registration, since no UI exists to bootstrap the first privileged user. After that, admins can assign roles through the user management interface.
</Tip>

## Model source

```csharp ApplicationUser.cs
using Microsoft.AspNetCore.Identity;
using System.ComponentModel.DataAnnotations;

namespace FisioCare_2.Models
{
    public class ApplicationUser : IdentityUser
    {
        public string FirstName { get; set; } = string.Empty;
        public string LastName { get; set; } = string.Empty;
        public string ImageFileName { get; set; } = string.Empty;
        public DateTime CreatedAt { get; set; }

        // Agregado para manejar el saldo de créditos del paciente
        [Range(0, int.MaxValue)]
        public int CreditosDisponibles { get; set; } = 0;

        public ICollection<Horario> Horarios { get; set; } = new List<Horario>();
    }
}
```
